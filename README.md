# E-commerce
Kafka 기반 비동기 이벤트 통신과 Spring Cloud 기반 MSA 아키텍처를 적용한
이커머스 토이 프로젝트입니다.

분산 환경에서 발생하는
서비스 간 통신, 동시성 제어, 데이터 정합성 문제를 직접 경험하고,
이를 구조적으로 해결하는 데 초점을 두었습니다.

## 🔧 기술 스택

| 분류 | 기술 | 설명 |
|------|------|------|
| Frontend | Vue.js, TypeScript | 사용자 화면 구성 |

#  문제 해결 사례

### 1. Kafka 기반 주문–재고 분산 트랜잭션 처리

**문제**  
주문 서비스와 상품(item) 서비스가 분산 환경에서 독립적으로 동작하면서,
주문 생성 후 재고 감소가 실패하거나,
재고 감소에 실패했음에도 주문이 정상 상태로 유지되는 등
주문–재고 간 데이터 정합성 문제가 발생할 수 있었습니다.

특히 Kafka 기반 비동기 통신 구조에서
이벤트 발행 실패나 서비스 장애가 발생할 경우,
재고 감소 또는 주문 실패 이벤트가 유실되어
상태 불일치가 고착되는 문제가 있었습니다.

**해결**  
이 문제를 해결하기 위해 Kafka 기반 이벤트 흐름과 SAGA 패턴을 적용하고,
이벤트 유실을 고려한 메시지 처리 구조를 함께 설계했습니다.

- 주문 서비스는 주문 생성 후 재고 감소 이벤트를 Kafka 토픽으로 발행
- 상품(item) 서비스는 해당 이벤트를 구독해 재고 차감을 처리
- 재고 부족 또는 처리 실패 시 주문 실패 이벤트를 발행해 주문 상태를 보상 처리
- 메시지 발행 과정에서 장애가 발생하더라도 이벤트가 유실되지 않도록, 이벤트를 DB에 안전하게 기록한 뒤 재처리 가능한 구조로 구성

**결과**  
- 주문–재고 간 처리 흐름이 이벤트 기반으로 분리되어 서비스 간 결합도가 낮아졌습니다.
- Kafka 발행 실패나 서비스 장애 상황에서도 이벤트 유실 없이 재처리가 가능해져 주문 상태와 재고 상태 간 정합성이 안정적으로 유지되었습니다.

### 2. 상품 수량 동시성 제어 (Spin Lock 적용)

**문제**  
여러 사용자가 동시에 동일한 상품을 구매하는 상황에서,
재고 차감 요청이 동시에 처리되며 중복 차감이 발생하는 Race Condition 문제가 확인되었습니다.

**해결**  
재고 차감 로직을 임계 구역으로 보호하기 위해 Spin Lock 기반 동시성 제어를 적용했습니다.
- 재고 감소 처리 전 락을 획득하도록 구성해,
- 동시에 여러 요청이 유입되더라도 한 번에 하나의 요청만 처리되도록 제어
- 락 획득 실패 시 즉시 재시도 또는 실패 처리하여 불필요한 대기와 중복 처리를 방지

**결과**  
- 동시 요청 환경에서도 재고 차감이 순차적으로 처리되어 중복 차감 및 수량 불일치 문제가 발생하지 않았습니다.
- 재고 처리에 대한 신뢰성과 일관성이 확보되었습니다.
